| Priority | **P1** |

### Diagnosis

Bots find our help site but can only reach the User Guide index/TOC page. The actual workflow creation steps (File > New > choose type > build in Analysis View) are behind iframe navigation that crawlers can't follow. The bot falls back to PyAnsys API docs because those are static HTML and easy to crawl. It also incorrectly labels Data workflows as "legacy."

### Source Topics to Update

- `modelcenter/mc-how-to-create-a-new-workflow.md`
- `modelcenter/mc-workflows.md`
- `modelcenter/mc-workflow-building-editing.md`
- `modelcenter/mc-choose-the-workflow-type.md`
- `modelcenter/mc-data-dependency-and-process-workflows.md`

### Proposed Fix

**Consolidate into a single self-contained workflow creation topic.** The current content is split across 5+ pages. Create or update one page that covers:
1. How to create a new workflow (File > New, choose Process or Data)
2. When to choose Process vs Data (brief comparison)
3. How to add components (Server Browser drag-and-drop)
4. How to link variables (Analysis View or Link Editor)
5. How to run it

This single page should be enough for a bot to answer "How do I create a workflow?" completely. The detailed deep-dives on schedulers, Loop/If components, etc. can stay as separate pages.

Additionally, verify that this topic's HTML on ansyshelp.ansys.com is directly reachable by crawlers (not behind an iframe-only path).

## T4: How do I set up sensitivity analysis in optiSLang?

| Field | Value |
|-------|-------|
| Product | optiSLang |
| Category | optimization |
| Verdict | **PARTIAL** |
| Priority | **P2** |
| Effort | small |
| Root Cause | crawlability |

### Diagnosis

Bots find our tutorials page and the PDF user guide, and return conceptually correct information (DOE > Metamodels > MOP with CoP, use the Sensitivity Wizard). But they miss the actionable UI steps: drag wizard from Wizards pane onto Scenery, configure the four wizard pages (Parametrize Inputs, Criteria, Sampling method, Additional options), Create MOP option. The Sensitivity Wizard topic exists in our docs but its step-by-step content isn't surfacing through search.

### Source Topics to Update

- `optislang/osl-sensitivity-wizard.md`
- `optislang/osl-sensitivity-analysis.md`

### Proposed Fix

**Add a concise step-by-step procedure to the Sensitivity Wizard topic.** The current topic describes the wizard's pages and options but may not have a clear numbered "do this, then this" procedure at the top. Add:

1. Open your project with a configured solver/process chain
2. In the Wizards pane, drag the **Sensitivity** wizard onto your process chain in the Scenery pane
3. **Parametrize Inputs:** select parameters with deterministic properties
4. **Criteria:** define responses, objectives, and constraints
5. **Sampling method:** choose from the decision tree (wizard disables inapplicable methods)
6. **Additional options:** enable **Create MOP** to generate a Metamodel of Optimal Prognosis node
7. Run the sensitivity study

This gives bots a single extractable procedure.
